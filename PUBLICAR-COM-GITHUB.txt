RONDA EDITORIAL — GITHUB + CLOUDFLARE WORKERS
==============================================

Esta é a versão indicada para quem não quer instalar Node.js e não quer colar
o código manualmente no editor do Cloudflare.

IMPORTANTE
----------

- Não use Direct Upload, Drag and drop ou Cloudflare Pages.
- O GitHub deve ser conectado a um CLOUDFLARE WORKER.
- Extraia este ZIP antes de enviar os arquivos ao GitHub.

PARTE 1 — CRIAR O REPOSITÓRIO NO GITHUB
----------------------------------------

1. Entre em https://github.com/ e clique em New repository.
2. Nome sugerido: ronda-editorial-webapp
3. O repositório pode ser público ou privado.
4. Clique em Create repository.
5. Na página do repositório, clique em uploading an existing file.
6. Extraia este pacote no Windows.
7. Arraste TODO o conteúdo da pasta extraída para o GitHub. Os arquivos
   package.json, package-lock.json e wrangler.jsonc devem ficar na raiz.
8. No campo de mensagem, escreva "Versão inicial" e clique em Commit changes.

PARTE 2 — CONECTAR O GITHUB AO CLOUDFLARE
------------------------------------------

9. No Cloudflare, abra Workers & Pages > Create application.
10. Em Import a repository, clique em Get started.
11. Autorize o GitHub e selecione o repositório ronda-editorial-webapp.
12. Use estas configurações:

      Production branch: main
      Root directory: deixe vazio
      Build command: npm run build:ui
      Deploy command: npx wrangler deploy

13. Clique em Save and Deploy.

O wrangler.jsonc já define:

- Worker: ronda-editorial-webapp
- Banco D1 com binding DB e provisionamento automático
- Ronda automática a cada cinco minutos
- Observabilidade ativada

PARTE 3 — PROTEGER A RONDA MANUAL (RECOMENDADO)
------------------------------------------------

14. No Worker publicado, abra Settings > Variables and Secrets.
15. Adicione um Secret:

      Nome: MANUAL_ROUND_TOKEN
      Valor: escolha uma chave longa e privada

16. Abra o painel da Ronda Editorial, entre em Ajustes e informe a mesma chave.

PARTE 4 — TESTAR
----------------

Abra, substituindo SEU-WORKER pelo endereço publicado:

https://SEU-WORKER.workers.dev/api/self-test
https://SEU-WORKER.workers.dev/api/health

Resultados esperados:

- self-test: "ok":true e "readWriteDelete":true
- health: "ready":true e "database":"connected"

Depois abra a página principal e clique em Executar ronda.

ATUALIZAÇÕES FUTURAS
--------------------

Quando uma nova versão for enviada ao branch main do GitHub, o Cloudflare fará
o build e o deploy automaticamente. Não será necessário usar o uploader.

SE QUISER CONECTAR UM WORKER JÁ EXISTENTE
-----------------------------------------

O nome no Cloudflare precisa ser igual ao campo "name" do wrangler.jsonc.
Esta versão usa "ronda-editorial-webapp". A maneira mais simples é importar o
repositório como um novo Worker. Se quiser reutilizar outro nome, altere o campo
"name" no wrangler.jsonc antes do primeiro deploy.
